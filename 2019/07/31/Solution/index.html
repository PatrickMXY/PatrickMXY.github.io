    NSNumber 修饰

    NSNumber 不可变的，即使使用copy修饰系统也并不会在使用时给他分配一块新的内存，为什么不用assign，弱指针在没有对象保持对他的使用他就会被释放，NSNumber 对象就会变成野指针，所以用strong修饰

    KeyChainItemWrapper使用的遇到的问题

    因为修改程序的bundleId 导致的，需要将程序的KeyChain Sharing中的bundle Id 修改成新的bundle id 底层原因暂未知

    打包问题

    证书缺失，可以通过Manage重新生成，不过这样会导致之前证书下发布的APP失效。最优解决方案是找到生成发布证书的源证书文件，导入到当前的设备钥匙串。
