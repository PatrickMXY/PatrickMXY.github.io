    • 这是一篇GitHub的使用理解

    日常使用SVN管理代码，并且通过Cornerstone管理，可以说是傻瓜式操作，但是GitHub作为最大的基佬交流网，怎么可以不加入其中呢

    1.注册GitHub账号

    这一步很重要，毕竟巧妇难为无米之炊

    2.新建仓库

    在自己的GitHub账号下新建仓库，选择 ** New Repositories**

    这里有一个许可证类型可能会让人懵逼，我们可以借助一张图来理解下

    想了解更多的可以参考阮一峰的博文开源许可证

    3.添加SSHKey

    SSHKey是什么的，顾名思义就是SSH的key，那SSH又是什么呢，讲道理我也不明白，可以理解成一种为了网络安全传输的网络协议，用于计算机之间的加密登录，感兴趣原理的话可以参考。SSH的原理

    添加流程：

    .打开终端 执行’ ssh-keygen -t rsa -b 4096 -C 你的邮箱 ‘,注意要填写自己的真是邮箱，这里我填的是GitHub的账号邮箱。（运行过程中有停顿，可以按空格带过，这样会在根目录下生成.ssh文件夹）

    .cd 到根目录下，执行’cat ~/.ssh/id_rsa.pub‘ 会得到一串东西，完整的复制下来，稍后会用到。

    .回到GitHub 的账号信息的Setting 会有一个新建SSHKey ,将刚才生成的一串内容复制到key里，title任意填写名称区分。至此我们生成SSHKey的任务是完成了，紧接着讲本机和GitHub绑定起来。（反正是让他们之间的安全通讯的桥梁建立起来，表述可能不严谨）

    .终端输入’ssh -T git@github.com‘ ，中途有提示你是否继续选择yes，成功后会有一段’You’ve successfully authenticated, but GitHub does not provide shell access’,至此表明你成功了。添加SSHKey流程结束

    Tips：

    1.每台电脑只需要一个SSHKey，他可以访问你的所有仓库

2.每个GitHub账号下可以生成多个SSHKey，例如你有多台设备，都想访问你的仓库内容

3.若SSHKey被删除了，重新生成一个替换之前就好

    4.提交

    终于来到我们的初衷了，在这之前我们需要先配置下我们的GitHub账户信息到本地

    终端执行’git config–globaluser.name your name’（表明提交代码的用户名）

    ‘git config–globaluser.email your_email@youremail.com‘(这里我填写的是GitHub账号的邮箱)

    代码提交流程：

    .首先我们第二步生成的仓库里clone下我们的地址，cd到项目管理路径下，执行’git clone your github repositories 地址’

    .成功执行后我们就clone该仓库到了本地文件夹，我们尝试在文件夹里做一些变更然后执行’git add .’将改动的地方添加的版本管理器

    .执行’git commit -m your log’ 提交到本地的版本控制库

    .执行’git push -u origin master’ 至此我们就可以成功的将变更内容提交到GitHub了，我们可以到我们的仓库下面查看更新内容

    多人维护的时候我们需要先’git pull’更新下，类似SVN需要先update 在 commit一样，避免代码冲突带来的麻烦

    5.小结

    GitHub管理相较SVN好像已经变成了大趋势，毕竟相较SVN我们用终端处理这些的时候显得逼格更胜一筹，当然也可以借助客户端，但是之前最好先用好终端命令，让我们知道它的一个流程的。
